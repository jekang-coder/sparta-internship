# 📋 인터뷰 트래커 서비스 기획서

> **"일경험 사업의 면접, 처음부터 끝까지 한 곳에서"**
> 참여 청년과 참여 기업이 면접 일정을 조율하고, 운영 매니저가 전체 현황을 한눈에 관리하는 채용 트래킹 서비스

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | 인터뷰 트래커 (Interview Tracker) |
| **목적** | 일경험 사업 참여 청년과 기업 간 면접 일정 조율 및 채용 절차 전반 관리 |
| **대상 사용자** | 일경험 사업 참여 청년(지원자), 참여 기업(면접관), 운영 매니저(어드민) |
| **접근 방식** | 카카오톡 소셜 로그인 (어드민은 @teamsparta.co 도메인 계정) |
| **런칭 형태** | 웹 서비스 (모바일 반응형) |

---

## 2. 문제 정의

일경험 사업의 면접 프로세스가 체계화되지 않아 발생하는 문제:

- **지원자(청년)**: 어느 기업에 언제 면접을 신청해야 할지 알 수 없음
- **기업(면접관)**: 지원자 관리, 일정 조율, 평가 기록이 파편화되어 있음
- **운영 매니저**: 전체 진행 현황을 실시간으로 파악하기 어렵고, 단계가 지연된 케이스를 놓침

**솔루션:** 기업이 면접 가능 일정과 방식을 등록하고 → 청년이 신청 → 기업이 승인 → 면접 진행 → 최종 합격자 선발까지 전 과정을 한 플랫폼에서 관리. 각 단계마다 카카오톡 안내 메시지로 누락 방지.

| 사용자 | 로그인 방식 | 주요 권한 |
|--------|------------|----------|
| **지원자(청년)** | 카카오톡 소셜 로그인 | 기업 면접 정보 조회, 면접 신청, 결과 확인 |
| **면접관(기업)** | 카카오톡 소셜 로그인 | 면접 일정 등록, 지원자 승인/거절, 평가 입력, 최종 합격자 선발 |

## 4. 핵심 기능

### 4-1. 기업 (면접관) 기능
- **면접 공고 등록**: 가능한 면접 일정(날짜/시간 슬롯), 면접 방식(대면/온라인), 모집 인원 등록
- **지원자 목록 조회**: 신청한 지원자 정보 확인 (이름, 연락처, 지원 동기 등)
- **면접 신청 승인/거절**: 지원자별 승인 처리 → 승인 시 면접 일정 자동 확정
- **평가 입력**: 면접 완료 후 평가 점수 + 코멘트 기록
- **최종 합격자 선발**: 평가 결과 기반으로 최종 합격 처리

### 4-2. 지원자 (청년) 기능
- **기업 면접 정보 조회**: 참여 기업 목록, 면접 일정/방식/모집 인원 확인
- **면접 신청**: 원하는 기업 + 면접 시간 슬롯 선택 후 신청
- **면접 일정 확인**: 승인된 면접 일정 확인
- **결과 확인**: 최종 합격/불합격 결과 확인

### 4-3. 카카오톡 알림 메시지
단계별 자동 알림 발송:

| 트리거 | 수신자 | 메시지 내용 |
|--------|--------|------------|
| 기업 면접 공고 등록 완료 | 매칭된 지원자 | "OO기업 면접 일정이 등록됐어요. 지금 바로 신청하세요!" |
| 지원자 신청 완료 | 기업 담당자 | "새로운 면접 신청이 들어왔어요. 확인 후 승인해주세요." |
| 기업 승인 완료 | 지원자 | "면접 일정이 확정됐어요! 일정을 확인해주세요." |
| 면접 D-1 | 지원자 + 기업 | "내일 면접이 있어요. 준비 잘 하셨나요?" |
| 면접 완료 후 | 기업 담당자 | "면접이 완료됐어요. 평가 점수를 기록해주세요." |
| 단계 미진행 알럿 | 지원자 or 기업 | "아직 [단계명] 단계를 완료하지 않으셨어요. 확인해주세요." |

### 4-4. 어드민 (운영 매니저) 기능
- **전체 현황 대시보드**: 단계별(신청/승인/면접예정/면접완료/합격자선발) 건수 한눈에 확인
- **기업별 / 지원자별 상세 현황**: 필터 + 검색으로 개별 케이스 추적
- **지연 케이스 알림**: 특정 단계에서 N일 이상 멈춘 케이스 하이라이트
- **알럿 발송 관리**: 단계별 카카오톡 알림 발송 현황 확인 및 수동 재발송

## 6. 기술 스택

| 영역 | 기술 | 역할 |
|------|------|------|
| **프론트엔드** | Next.js 14 (App Router), TypeScript, Tailwind CSS | UI/UX, SSR |
| **백엔드** | Nest.js, TypeScript | API 서버, 카카오 연동 |
| **데이터베이스** | Supabase (PostgreSQL) | 지원자/기업/면접 데이터 저장 |
| **인증** | 카카오 소셜 로그인 (OAuth 2.0) | 사용자 인증 |
| **알림** | 카카오 알림톡 API | 단계별 자동 메시지 발송 |
| **배포** | Vercel | 프론트 + 백엔드 배포 |
| **버전 관리** | GitHub | 코드 관리 |

### 규모 및 인프라 고려사항

- **예상 동시 접속자**: 최대 100명 (카카오톡 로그인 기준)
- **Supabase**: Free/Pro 플랜으로 충분 (커넥션 풀 기본 설정으로 처리 가능)
- **Vercel**: Serverless Functions 자동 스케일링으로 트래픽 대응
- **카카오 알림톡**: 동시 발송 부하 고려 → 단계 트리거 시 큐 없이 직접 발송 가능한 규모
- **DB 인덱스 필수**: `applications.status`, `applications.applicant_id`, `interview_slots.company_id` 컬럼 인덱싱으로 조회 성능 확보

## 7. 데이터 구조 (Supabase)

### users 테이블
```sql
CREATE TABLE users (
  id            UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  kakao_id      VARCHAR(100) UNIQUE NOT NULL,  -- 카카오 고유 ID
  email         VARCHAR(200),                   -- @teamsparta.co → 어드민 판별
  name          VARCHAR(100) NOT NULL,
  role          VARCHAR(20) NOT NULL,           -- 'applicant' | 'interviewer' | 'admin'
  phone         VARCHAR(20),
  created_at    TIMESTAMP DEFAULT NOW()
);
```

### companies 테이블
```sql
CREATE TABLE companies (
  id            UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id       UUID REFERENCES users(id),     -- 면접관 계정
  name          VARCHAR(200) NOT NULL,
  description   TEXT,
  created_at    TIMESTAMP DEFAULT NOW()
);
```

### interview_slots 테이블
```sql
CREATE TABLE interview_slots (
  id            UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  company_id    UUID REFERENCES companies(id),
  interview_date DATE NOT NULL,
  start_time    TIME NOT NULL,
  end_time      TIME NOT NULL,
  method        VARCHAR(20) NOT NULL,          -- 'offline' | 'online'
  location      VARCHAR(300),                  -- 대면 시 장소 or 온라인 링크
  capacity      INT DEFAULT 1,                 -- 슬롯당 모집 인원
  created_at    TIMESTAMP DEFAULT NOW()
);
```

### applications 테이블
```sql
CREATE TABLE applications (
  id            UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  applicant_id  UUID REFERENCES users(id),
  slot_id       UUID REFERENCES interview_slots(id),
  status        VARCHAR(30) NOT NULL DEFAULT 'pending',
                -- 'pending' | 'approved' | 'rejected' | 'completed' | 'passed' | 'failed'
  score         INT,                           -- 면접 평가 점수 (0~100)
  comment       TEXT,                          -- 면접관 코멘트
  applied_at    TIMESTAMP DEFAULT NOW(),
  updated_at    TIMESTAMP DEFAULT NOW()
);
```

### notifications 테이블
```sql
CREATE TABLE notifications (
  id            UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id       UUID REFERENCES users(id),
  application_id UUID REFERENCES applications(id),
  type          VARCHAR(50) NOT NULL,          -- 알림 유형 (예: 'slot_open', 'approved' 등)
  message       TEXT NOT NULL,
  sent_at       TIMESTAMP,
  status        VARCHAR(20) DEFAULT 'pending', -- 'pending' | 'sent' | 'failed'
  created_at    TIMESTAMP DEFAULT NOW()
);
```

## 11. 성공 지표 (KPI)

- **전환율**: 로그인 후 면접 신청까지 완료 비율 70% 이상

- **알림 효과**: 카카오톡 알림 수신 후 24시간 내 단계 진행 비율 80% 이상
- **운영 효율**: 어드민 개입 없이 자동 처리되는 단계 비율 80% 이상
